## 5. Логистическая регрессия

### 5.1. Что такое логистическая регрессия

Логистическая регрессия — модель **бинарной классификации**, предсказывающая вероятность принадлежности к классу:

```
P(y=1|x) = σ(b₀ + b₁x₁ + ... + bₙxₙ)
```

где σ(z) = 1 / (1 + e^(-z)) — **сигмоида**, преобразующая любое число в диапазон (0, 1).

### 5.2. Гиперпараметры базовой модели

## 6. Кросс-валидация

Использована **5-fold Stratified Cross-Validation** на обучающей выборке:

- Stratified — пропорции классов сохраняются в каждом фолде
- 5 фолдов — данные делятся на 5 частей, модель обучается на 4, проверяется на 1

Результаты по фолдам подтверждают стабильность модели.

## 9. Precision vs Recall

### 9.1. Определения

- **Precision** = TP / (TP + FP) — из всех предсказанных положительных, сколько реально положительны
- **Recall** = TP / (TP + FN) — из всех реально положительных, скольких мы нашли

### 9.2. Что важнее в нашей задаче

В контексте **классификации качества вина**:

| Ошибка | Что означает | Последствия |
|---|---|---|
| **FP** (обычное → хорошее) | Покупатель переплатит за посредственное вино | Потеря доверия, репутационный ущерб |
| **FN** (хорошее → обычное) | Хорошее вино продаётся дешевле | Упущенная прибыль, но не критично |

**Вывод: Precision важнее Recall.** Лучше упустить часть хороших вин, чем рекомендовать обычное вино как хорошее — это подрывает доверие покупателей.

## 10. Регуляризация

### 10.1. Зачем нужна

Регуляризация добавляет штраф за большие коэффициенты, предотвращая переобучение. Параметр **C** — обратная сила регуляризации: маленький C = сильная регуляризация.

### 10.2. Типы

- **L2 (Ridge):** `Σ(bᵢ²)` — уменьшает коэффициенты, но не обнуляет
- **L1 (Lasso):** `Σ|bᵢ|` — может обнулять коэффициенты (отбор признаков)
- **ElasticNet:** комбинация L1 + L2 (`l1_ratio = 0.5`)

### 10.3. Подбор C

Для каждого типа перебраны значения C от 10⁻³ до 10³ с 5-fold кросс-валидацией. Построен график зависимости accuracy от C.

### 10.4. Результаты

Сравнительная таблица лучших моделей приведена в ноутбуке. Ключевое отличие L1: количество ненулевых коэффициентов может быть меньше 11 — модель автоматически отбрасывает неинформативные признаки.

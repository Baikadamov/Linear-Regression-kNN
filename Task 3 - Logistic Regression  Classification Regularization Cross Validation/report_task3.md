## 5. Логистическая регрессия

### 5.1. Что такое логистическая регрессия

Логистическая регрессия — это модель **бинарной классификации**. В отличие от линейной регрессии, которая предсказывает число, логистическая предсказывает **вероятность** принадлежности к классу.

Формула:

```
P(y=1|x) = σ(b₀ + b₁x₁ + ... + bₙxₙ)
```

где σ(z) = 1 / (1 + e^(-z)) — **сигмоида**, преобразующая любое число в диапазон (0, 1).

### 5.2. Гиперпараметры

| Параметр | Значение | Пояснение |
|---|---|---|
| `C` | 1.0 (по умолчанию) | Обратная сила регуляризации |
| `penalty` | 'l2' (по умолчанию) | Тип регуляризации |
| `max_iter` | 10000 | Максимум итераций для сходимости |
| `random_state` | 42 | Воспроизводимость |

### 5.3. Результаты базовой модели

Accuracy на тестовой выборке: **~0.97** (97%)

## 6. Кросс-валидация

Использована **5-fold Stratified Cross-Validation** на обучающей выборке:

- Stratified — пропорции классов сохраняются в каждом фолде
- 5 фолдов — данные делятся на 5 частей, модель обучается на 4, проверяется на 1, повторяется 5 раз

**Результат:** accuracy ~0.97 (+/- ~0.02)

Стабильные результаты по фолдам подтверждают, что модель не переобучена и хорошо обобщает.

## 7. Оптимальный порог (threshold)

### 7.1. Что такое порог

По умолчанию логистическая регрессия использует порог 0.5: если P(y=1) ≥ 0.5, предсказываем класс 1, иначе — класс 0. Но этот порог не всегда оптимален.

### 7.2. Методика

Перебраны значения порога от 0.05 до 0.95 с шагом 0.01. Для каждого порога рассчитаны Precision, Recall и F1-score.

### 7.3. Результат

Лучший порог по F1-score определён автоматически. При изменении порога видно характерное поведение:
- При понижении порога Recall растёт, но Precision падает
- При повышении порога — наоборот
- F1 находит баланс между ними

Построен также сравнительный confusion matrix при стандартном (0.5) и оптимальном пороге.

## 8. Precision vs Recall

### 8.1. Определения

- **Precision** = TP / (TP + FP) — из всех, кого мы назвали положительными, сколько реально положительны
- **Recall** = TP / (TP + FN) — из всех реально положительных, скольких мы нашли

### 8.2. Что важнее в нашей задаче

В контексте **диагностики рака молочной железы**:

| Ошибка | Что означает | Последствия |
|---|---|---|
| **FN (пропуск рака)** | Больной пациент признан здоровым | Отсутствие лечения → угроза жизни |
| **FP (ложная тревога)** | Здоровый пациент направлен на доп. обследование | Стресс и расходы, но не опасно |

**Вывод: Recall важнее Precision.** Цена пропущенного рака (FN) несопоставимо выше, чем цена ложной тревоги (FP). Лучше отправить здорового на дополнительные анализы, чем пропустить рак.

Для максимизации recall можно понизить порог классификации, что увеличит чувствительность модели ценой некоторого снижения precision.

## 9. Регуляризация

### 9.1. Зачем нужна регуляризация

Регуляризация добавляет штраф за большие коэффициенты в функцию потерь, предотвращая переобучение. В логистической регрессии параметр **C** — это **обратная** сила регуляризации: маленький C = сильная регуляризация, большой C = слабая.

### 9.2. L2-регуляризация (Ridge)

```
Потери = Log-loss + (1/C) × Σ(bᵢ²)
```

- Уменьшает все коэффициенты, но **не обнуляет** их
- Хорошо работает при мультиколлинеарности
- Все 30 признаков остаются в модели

### 9.3. L1-регуляризация (Lasso)

```
Потери = Log-loss + (1/C) × Σ|bᵢ|
```

- Может **обнулять** коэффициенты → автоматический отбор признаков (feature selection)
- При сильной регуляризации (маленький C) в модели остаются только самые важные признаки

### 9.4. ElasticNet (L1 + L2)

```
Потери = Log-loss + (1/C) × (l1_ratio × Σ|bᵢ| + (1-l1_ratio) × Σ(bᵢ²))
```

- Комбинация обоих штрафов
- `l1_ratio = 0.5` использован по умолчанию

### 9.5. Подбор параметра C

Для каждого типа регуляризации перебраны 20 значений C от 10⁻⁴ до 10⁴ (логарифмическая шкала) с 5-fold кросс-валидацией.

### 9.6. Результаты

Сравнительная таблица лучших моделей (на тестовой выборке):

| Модель | C | Accuracy | Precision | Recall | F1 | AUC-ROC | Ненулевых коэф. |
|---|---|---|---|---|---|---|---|
| L2 (Ridge) | — | ~0.97 | ~0.97 | ~0.99 | ~0.98 | ~0.99 | 30 |
| L1 (Lasso) | — | ~0.97 | ~0.97 | ~0.99 | ~0.98 | ~0.99 | <30 |
| ElasticNet | — | ~0.97 | ~0.97 | ~0.99 | ~0.98 | ~0.99 | <30 |

**Примечание:** точные значения приведены в ноутбуке.

Ключевое отличие L1: количество ненулевых коэффициентов меньше 30 — модель автоматически отбросила неинформативные признаки.

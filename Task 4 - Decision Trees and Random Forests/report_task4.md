## 3. Part 1 — Классификация

### 3.1. EDA и предобработка

- **Пропуски:** отсутствуют
- **Распределение классов:** три класса распределены относительно равномерно (59/71/48), стратификация при разделении сохраняет пропорции
- **Корреляции:** наибольшая корреляция между flavanoids, total_phenols, OD280/OD315; наибольшая связь с классом у flavanoids, proline, color_intensity
- **Разделение:** `train_test_split(test_size=0.2, random_state=42, stratify=y)` — 142 train, 36 test
- **Масштабирование:** StandardScaler (fit на train, transform на test) — для сравнимости, хотя деревья не требуют масштабирования

### 3.2. Decision Tree

**Что это:** алгоритм, который принимает решения путём последовательного разделения данных по признакам. На каждом узле выбирается признак и порог, дающие максимально «чистое» разделение (по критерию Gini).

**Pruning (обрезка):** ограничение роста дерева для предотвращения переобучения. Подбираемые параметры:
- `max_depth` — максимальная глубина дерева
- `min_samples_split` — минимальное количество примеров для разделения узла
- `min_samples_leaf` — минимальное количество примеров в листе

**Подбор гиперпараметров:** 5-fold кросс-валидация с перебором `max_depth` от 1 до 15. Оптимальная глубина выбрана по максимальному среднему accuracy на валидационных фолдах.

**Визуализация:** построено дерево решений с помощью `plot_tree` — наглядно показывает правила разделения.

**Результаты:** confusion matrix и classification report показывают высокую точность классификации. ROC-кривые построены для каждого класса (OvR подход) и macro-average.

### 3.3. Random Forest

**Что это:** ансамбль деревьев решений. Каждое дерево обучается на случайной подвыборке данных (bagging) и случайном подмножестве признаков. Финальное решение — голосование.

**Bagging (Bootstrap Aggregating):** для каждого дерева создаётся случайная выборка с повторениями из обучающих данных. Это делает деревья разными и снижает переобучение.

**Подбор гиперпараметров:**
- `n_estimators` (количество деревьев): перебор значений 50, 100, 200, 300, 500 через 5-fold CV
- `max_depth`: перебор значений от 1 до 15 через 5-fold CV

**Результаты:** Random Forest показывает более стабильные и точные результаты, чем одиночное дерево. Confusion matrix и classification report подтверждают высокое качество.

### 3.4. Оптимальный порог

Для мультиклассовой задачи оптимальный порог определяется **для каждого класса отдельно**. Для каждого класса перебираются значения порога от 0.05 до 0.95, и для каждого рассчитываются precision, recall и F1-score (OvR подход). Лучший порог выбирается по максимальному F1.

### 3.5. Precision vs Recall

В контексте **виноделия** (классификация сортов вина):

| Ошибка | Что означает | Последствия |
|---|---|---|
| **FP (ложная классификация)** | Дешёвое вино отнесли к дорогому сорту | Репутационные и финансовые потери |
| **FN (пропуск класса)** | Дорогое вино отнесли к дешёвому | Упущенная прибыль |

**Вывод: Precision важнее Recall.** Неправильная классификация сорта вина (назвать дешёвое вино дорогим) наносит больший репутационный ущерб, чем недооценка качества. Поэтому важнее минимизировать ложные срабатывания (FP).

### 3.6. Feature Importance

Для обеих моделей (Decision Tree и Random Forest) рассчитана важность признаков через `feature_importances_`. Построен сравнительный barplot. Определён самый значимый признак.

## 7. Выводы

1. **Деревья решений** — интерпретируемая модель, но склонны к переобучению. Pruning (ограничение глубины, min_samples) значительно улучшает обобщающую способность.

2. **Random Forest** превосходит одиночное дерево в обеих задачах за счёт ансамблирования (bagging). Метод устойчив к переобучению и даёт стабильные результаты.

3. **Кросс-валидация** (5-fold) позволила объективно подобрать оптимальные гиперпараметры: глубину дерева и количество деревьев в лесу.

4. **Precision vs Recall:** в задаче классификации вина precision важнее — неправильная классификация сорта наносит репутационный ущерб.

5. **Feature Importance:** деревья автоматически определяют значимость признаков. В классификации вина наиболее информативны flavanoids/proline/color_intensity. В регрессии стоимости жилья — MedInc (медианный доход).

6. **Деревья vs линейные модели:** деревья лучше улавливают нелинейные зависимости и взаимодействия между признаками. California Housing — хороший пример, где нелинейная модель (Random Forest) превосходит линейную регрессию.

# Отчёт по лабораторной работе: Предсказание стоимости жилья в Калифорнии

## 1. Цель работы

Построить модель машинного обучения для предсказания медианной стоимости жилья (`median_house_value`) на основе характеристик районов Калифорнии. Сравнить два подхода: **линейную регрессию** и **случайный лес (Random Forest)**.

## 2. Датасет

Используется датасет **California Housing** (файл `housing.csv`) — классический набор данных из переписи населения Калифорнии 1990 года.

- **Объём:** 20 640 записей, 10 признаков
- **Целевая переменная:** `median_house_value` — медианная стоимость жилья в районе (в долларах)

### Описание признаков

| Признак | Описание |
|---|---|
| `longitude`, `latitude` | Координаты района |
| `housing_median_age` | Медианный возраст домов |
| `total_rooms` | Общее количество комнат в районе |
| `total_bedrooms` | Общее количество спален в районе |
| `population` | Население района |
| `households` | Количество домохозяйств |
| `median_income` | Медианный доход (в десятках тысяч долларов) |
| `ocean_proximity` | Близость к океану (категориальный: NEAR BAY, NEAR OCEAN, INLAND, <1H OCEAN, ISLAND) |

## 3. Предобработка данных

### 3.1. Обработка пропусков

В столбце `total_bedrooms` обнаружено **207 пропущенных значений** (из 20 640). Пропуски заполнены **медианой** — это стандартный подход, устойчивый к выбросам (в отличие от среднего).

### 3.2. Кодирование категориальных признаков

Столбец `ocean_proximity` (текстовый) преобразован в числовой формат с помощью **One-Hot Encoding** (`pd.get_dummies`). Параметр `drop_first=True` убирает одну категорию (`<1H OCEAN`), чтобы избежать мультиколлинеарности (ловушка dummy-переменных).

### 3.3. Feature Engineering (создание новых признаков)

Из исходных данных созданы три более информативных признака:

| Новый признак | Формула | Зачем |
|---|---|---|
| `rooms_per_household` | `total_rooms / households` | Среднее число комнат на домохозяйство — характеризует размер жилья |
| `bedrooms_per_room` | `total_bedrooms / total_rooms` | Доля спален от общего числа комнат — косвенно отражает тип жилья |
| `population_per_household` | `population / households` | Среднее число жителей на домохозяйство — плотность заселения |

После создания новых признаков исходные абсолютные столбцы (`total_rooms`, `total_bedrooms`, `population`, `households`) были **удалены**, так как относительные признаки более информативны для модели.

### 3.4. Удаление выбросов

Записи со значением `median_house_value = 500 000` (максимальный cap в данных) были удалены. Это ограничение датасета — значения >= 500 001 обрезаны, что создаёт искусственный пик на гистограмме и искажает обучение модели.

### 3.5. Стандартизация (для линейной регрессии)

Перед обучением линейной регрессии признаки отмасштабированы с помощью `StandardScaler` (среднее = 0, стд. отклонение = 1). Это нужно потому, что линейная регрессия чувствительна к масштабу признаков — без стандартизации признаки с большими значениями (например, координаты) доминировали бы над признаками с малыми (например, `median_income`).

## 4. Визуализация данных

### 4.1. Гистограмма распределения стоимости жилья

Показывает, как распределены цены. До удаления выбросов виден выраженный пик на отметке 500 000 (искусственное ограничение). После удаления распределение становится более естественным.

### 4.2. Матрица корреляции (Heatmap)

Показывает попарные корреляции между всеми числовыми признаками. Ключевой вывод: **`median_income` имеет наибольшую корреляцию с ценой жилья** — это главный предиктор.

### 4.3. Scatter plot: доход vs стоимость

Наглядно показывает положительную зависимость между медианным доходом и ценой жилья. Чем выше доход в районе, тем дороже жильё.

## 5. Модель 1: Линейная регрессия

### Что это

Линейная регрессия строит линейную зависимость вида:

```
y = b₀ + b₁x₁ + b₂x₂ + ... + bₙxₙ
```

Каждому признаку присваивается коэффициент (вес), показывающий его вклад в предсказание.

### Разделение данных

- **80%** — обучающая выборка
- **20%** — тестовая выборка
- `random_state=42` — фиксированное зерно для воспроизводимости

### Результаты

| Метрика | Значение | Что означает |
|---|---|---|
| **MAE** | 46 974.86 | В среднем модель ошибается на ~47 000 $ |
| **RMSE** | 63 344.28 | С учётом крупных ошибок — ~63 000 $ |
| **R²** | 0.5815 | Модель объясняет ~58% дисперсии целевой переменной |

### Анализ коэффициентов

Наиболее важные признаки (по модулю коэффициента после стандартизации):

1. **median_income** (+61 029) — главный положительный фактор
2. **longitude / latitude** (~ -48 000 / -46 000) — географическое расположение существенно влияет на цену
3. **ocean_proximity_INLAND** (-18 372) — удалённость от побережья снижает цену

### Вывод

R² = 0.58 — модель объясняет чуть больше половины вариации. Линейная регрессия **не может уловить нелинейные зависимости** в данных, что ограничивает её точность.

## 6. Модель 2: Random Forest (Случайный лес)

### Что это и зачем

Random Forest — ансамблевый метод, строящий множество решающих деревьев и усредняющий их предсказания. Выбран для сравнения с линейной регрессией, потому что:

- Улавливает **нелинейные зависимости**
- Устойчив к выбросам
- Не требует стандартизации признаков

### Гиперпараметры

| Параметр | Значение | Пояснение |
|---|---|---|
| `n_estimators` | 300 | Количество деревьев в ансамбле |
| `max_depth` | None | Деревья растут до полного разделения |
| `min_samples_split` | 2 | Минимум 2 образца для разбиения узла |
| `min_samples_leaf` | 1 | Минимум 1 образец в листе |
| `n_jobs` | -1 | Использовать все ядра процессора |

### Результаты

| Метрика | Значение | Что означает |
|---|---|---|
| **MAE** | 30 543.00 | В среднем ошибка ~30 500 $ |
| **RMSE** | 46 095.94 | С учётом крупных ошибок — ~46 000 $ |
| **R²** | 0.7784 | Модель объясняет ~78% дисперсии |

## 7. Сравнение моделей

| Метрика | Линейная регрессия | Random Forest | Улучшение |
|---|---|---|---|
| **MAE** | 46 975 | 30 543 | **-35%** |
| **RMSE** | 63 344 | 46 096 | **-27%** |
| **R²** | 0.5815 | 0.7784 | **+34%** |

Random Forest значительно превосходит линейную регрессию по всем метрикам. Это ожидаемо: зависимость стоимости жилья от признаков **нелинейна**, и древовидная модель лучше её моделирует.

### График «реальные vs предсказанные»

Для обеих моделей построен scatter plot: идеальная модель дала бы все точки на диагонали (красная пунктирная линия). У Random Forest точки плотнее прилегают к диагонали, что визуально подтверждает более высокую точность.
